This screen — Edit post (/admin/posts/{id}/edit)

  • Edits every field of an existing post, prefilled (§4.1).
  • Same layout as "New post" but with values filled in.
  • A "Delete this post" button sits below, separated by a top border.
  • On save, a published post redirects to its public page; a Hangul slug must be encodeURIComponent-encoded in the redirect to avoid a header error (the earlier 500 cause).
